# 数据管理插件

## 介绍

对咸鱼云网盘的数据进行统一管理，进行失效数据识别与修复、数据视察、清理、备份、文件元数据识别、未知文件类型识别功能。

## 功能

## 1. 失效数据识别与修复

### 失效数据定义

由于咸鱼云的主文件系统采用文件记录 + 文件物理存储的方式，由文件记录维护用户的文件结构。当文件记录丢失时，原有的文件物理存储就会成为失效数据导致无法再被正常访问到。

### 失效数据识别

识别方式取决于存储模式(SysCommonConfig#storeMode)

| 存储模式   | 识别方式                                           |
|--------|------------------------------------------------|
| UNIQUE | 通过文件物理存储的唯一标识符（如哈希值）与文件记录进行比对，找出没有对应文件记录的物理存储。 |
| RAW    | 通过扫描存储系统中的所有物理存储，检查每个物理存储是否有对应的文件记录。           |

### 失效数据修复

| 存储模式   | 修复方式                                                      |
|--------|-----------------------------------------------------------|
| UNIQUE | 由于已丢失文件记录无法恢复文件原有的文件名、所属用户、存储路径，无法自动修复，需要用户手动处理，决定删除还是保留。 |
| RAW    | 通过分析物理存储的内容和元数据，用户确认后可直接重建文件记录，恢复文件的访问路径和相关信息。            |

### 失效数据状态机

"处理完成"作为唯一终态，通过`处理方式`字段（丢弃/认领/自动修复）区分完成原因。

**UNIQUE模式：**

```
待处理 --发布--> 已发布可认领 --认领--> 已认领 --标记完成--> 处理完成
  |                 |                    |
  |                 | (取消发布)          |
  |<----------------|                    |
  |                                      |
  +---丢弃-----------------------------> 处理完成(丢弃)
```

**RAW模式：**

```
待处理 --快速修复成功--> 处理完成(自动修复)
  |
  +---丢弃-----------> 处理完成(丢弃)
```

### 存储模式操作矩阵

| 操作 | RAW | UNIQUE |
|------|-----|--------|
| 快速修复 | 支持 | 不支持 |
| 认领 | 不支持 | 支持（多次） |
| 发布为可认领 | 不适用 | 支持 |
| 取消发布 | 不适用 | 支持（无认领记录时） |
| 丢弃（物理存储） | 不适用 | 支持（引用计数=0且未被认领） |
| 丢弃（文件记录） | 支持 | 不适用 |
| 标记处理完成 | 修复成功时自动 | 管理员手动 |
| 文件识别 | 支持 | 支持 |

### 执行流程

1. 失效数据检测
  由管理员主动发起检测扫描，通过异步任务执行，根据存储模式选择相应的识别方式，扫描存储系统，找出失效数据并保存明细。失效数据检测实现见文档[invalid-data-detect.md](./docs/invalid-data-detect.md)。
  检测执行策略：
    - 并发控制：限制只能同时有一个进行中的检测或文件类型识别任务
    - 结果清理：检测开始时只清除上一轮"待处理"状态的记录，保留"处理完成"的记录作为历史
    - 任务重试：失败后保留已有结果，重试时清除未完成结果并重新全量扫描

  失效数据检测完成后，管理员的下一步可选择的动作：
    - 数据浏览：可以查看失效文件与失效文件记录的详细信息，包括物理存储路径、文件大小、最后修改时间、是否为待识别文件、文件类型等。
    - 文件识别：针对需要识别的文件（UNIQUE模式下为待识别文件），创建异步任务执行文件识别操作，提取文件的元数据（如图片的宽高、视频的时长等），识别完成后将识别结果更新到失效数据记录对应字段，以辅助管理员判断该文件的内容和价值。
    - 执行快速修复（仅RAW模式）
      - 失效文件记录：检查物理存储是否存在，若存在且原路径可用则重建文件记录；若原路径不可用则修复失败；若物理存储不存在则通过md5在全网查找相同文件并复制，否则修复失败
      - 失效物理存储：重建该存储对应的文件记录，恢复文件访问路径和相关信息
      - 修复成功后自动标记为"处理完成"（处理方式：自动修复）
    - 执行数据丢弃：对于确认不需要保留的失效数据，执行数据丢弃操作
      - UNIQUE模式：删除物理存储文件（前置条件：MD5引用计数=0且状态不为"已认领"）
      - RAW模式：删除失效文件记录（前置条件：状态为"待处理"）
    - 发布为可认领（仅UNIQUE模式）：管理员将失效物理存储发布为公开可认领状态
    - 取消发布（仅UNIQUE模式）：取消发布，回到"待处理"状态（前置条件：无认领记录）
    - 标记处理完成（仅UNIQUE模式）：管理员手动标记处理完成，关闭认领通道（前置条件：状态为"已认领"）
2. 数据认领（针对UNIQUE模式下失效物理存储）
  认领权限规则（使用UIDValidator校验）：
    - 失效数据未发布为公共可认领：仅管理员可操作
    - 失效数据已发布为公共可认领：任何用户可认领，但targetUid必须为当前用户uid；管理员无此限制
    - targetUid为0（公共网盘）：仅管理员可操作
  认领流程：
    - 指定文件名、保存路径（在公共网盘/我的网盘下，公共网盘只有管理员能选择），根据文件名+路径，重建一条指向该物理存储的文件记录
    - 文件名+路径冲突时拒绝操作并提示用户修改
    - 文件可被多次认领，每次认领创建一条认领记录（继承AuditModel，包含invalid_data_id、target_uid、file_name、save_path字段）
    - 认领后失效数据记录状态更新为"已认领"，但仍允许继续认领
    - 已认领的失效数据不允许丢弃

## 文件识别

采取读取文件头、尝试读取某些特定类型文件的元数据的方式进行类型识别。提供以下类型的文件识别：

| 文件类型 | 可能的后缀名                          | 元数据                    |
|------|---------------------------------|------------------------|
| 图片   | jpg、jpeg、png、bmp、gif、tiff等      | 图片宽高、拍摄时间等             |
| 视频   | mp4、avi、mkv等                    | 视频时长、分辨率等              |
| 音频   | mp3、wav、flac等                   | 音频时长、比特率、专辑名、歌手、标题、备注等 |
| 文档   | pdf、doc、docx、xls、xlsx、ppt、pptx等 | 文档页数、作者等               |
| 压缩文件 | zip、rar、7z、tar、gz等              | 压缩文件内包含的文件列表(提取部分)     |
| 镜像文件 | img、iso等                        | 镜像内包含的文件列表(提取部分)       |
| 文本文件 | txt、log等                        | 文件编码、行数、可能的程序代码语言      |

文件识别的设计方案见 [file-type-check.md](./docs/file-type-check.md)
